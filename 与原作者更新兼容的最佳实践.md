# 与原作者更新兼容的最佳实践

本指南帮助你在添加自定义壁纸的同时，保持与原作者仓库更新的兼容性，确保你可以继续同步原作者的最新壁纸。

## 1. 保持仓库结构

### 1.1 核心目录结构

```
├── .github/           # GitHub Actions 配置（不要修改）
├── bing/              # Bing 壁纸（由原作者维护）
├── data/              # 前端数据文件（自动生成）
├── metadata/          # 元数据文件（自动生成）
├── preview/           # 预览图（自动生成）
├── scripts/           # 处理脚本（不要修改）
├── thumbnail/         # 缩略图（自动生成）
├── wallpaper/         # 壁纸文件（可以添加自定义壁纸）
├── timestamps-backup-all.txt  # 时间戳文件（自动更新）
└── stats.json         # 统计信息（自动更新）
```

### 1.2 不要修改的文件

- `.github/workflows/sync-all.yml` - 同步工作流配置
- `scripts/` 目录下的所有脚本文件
- `bing/` 目录（由原作者维护）
- `data/` 目录（自动生成）
- `metadata/` 目录（自动生成）
- `preview/` 目录（自动生成）
- `thumbnail/` 目录（自动生成）
- `stats.json` 文件（自动生成）

## 2. 添加自定义壁纸的最佳实践

### 2.1 目录组织

- **推荐**：在 `wallpaper/[series]/` 下创建独立的分类目录来存放自定义壁纸
- **例如**：
  - `wallpaper/desktop/自定义/`
  - `wallpaper/mobile/自定义/`
  - `wallpaper/avatar/自定义/`

### 2.2 命名规范

- 使用描述性文件名，避免与原作者的壁纸重名
- 支持的格式：JPG、PNG、WebP、GIF
- 文件名建议使用中文或英文，避免特殊字符

### 2.3 处理流程

1. **上传壁纸**：将壁纸上传到对应的分类目录
2. **运行本地处理脚本**：执行 `local-process.ps1` 来更新时间戳和生成前端数据
3. **提交更改**：将更改提交到你的 fork 仓库
4. **同步原作者更新**：定期从原作者仓库同步最新更新

## 3. 同步原作者更新

### 3.1 手动同步步骤

1. **添加原作者仓库作为上游**：
   ```bash
   git remote add upstream https://github.com/IT-NuanxinPro/wallpaper-gallery.git
   ```

2. **拉取原作者更新**：
   ```bash
   git fetch upstream
   git merge upstream/main
   ```

3. **解决冲突**：如果有冲突，手动解决冲突

4. **推送更新**：
   ```bash
   git push
   ```

### 3.2 自动同步

你可以设置 GitHub Actions 来自动同步原作者的更新：

1. 在 `.github/workflows/` 目录下创建 `sync-upstream.yml` 文件
2. 配置自动同步工作流

## 4. 常见问题与解决方案

### 4.1 同步冲突

**问题**：同步原作者更新时出现冲突

**解决方案**：
- 手动解决冲突，保留原作者的核心文件
- 确保你的自定义壁纸在独立的目录中
- 使用 `git merge --strategy-option theirs` 优先保留原作者的更改

### 4.2 脚本执行失败

**问题**：本地处理脚本执行失败

**解决方案**：
- 确保安装了 Node.js 14+ 和 Git
- 检查目录结构是否正确
- 查看错误信息，针对性解决问题

### 4.3 网站不显示新壁纸

**问题**：上传壁纸后网站不显示

**解决方案**：
- 确保运行了本地处理脚本
- 清除浏览器缓存
- 检查网络连接和 CDN 缓存
- 等待几分钟让网站缓存更新

## 5. 性能优化

### 5.1 图片优化

- **分辨率**：根据系列选择合适的分辨率
  - Desktop：1920x1080 或更高
  - Mobile：1080x1920 或更高
  - Avatar：512x512 或更高

- **文件大小**：建议控制在 10MB 以内
- **格式**：优先使用 WebP 格式，其次是 JPG

### 5.2 目录结构优化

- 避免创建过深的目录结构
- 合理分类，便于管理和查找
- 定期清理无效或重复的壁纸

## 6. 安全建议

- **不要上传**：版权受限、色情、暴力等违规内容
- **检查文件**：确保上传的文件不包含恶意代码
- **隐私保护**：不要上传包含个人信息的图片
- **备份**：定期备份你的自定义壁纸

## 7. 总结

通过遵循以上最佳实践，你可以：

1. **添加自定义壁纸**：按照规范组织和上传你的壁纸
2. **保持更新兼容**：定期同步原作者的最新壁纸
3. **确保网站正常运行**：正确处理 metadata 和前端数据
4. **优化性能**：合理组织目录结构和优化图片

这样，你就可以拥有一个既包含原作者优质壁纸，又包含你个人收藏的完整壁纸网站。
