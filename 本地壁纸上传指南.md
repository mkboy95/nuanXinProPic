# 本地壁纸上传指南

本指南帮助你在自己 fork 的图床仓库中添加自定义壁纸，同时保持与原作者更新的兼容性。

## 目录结构

图床仓库的壁纸存储在以下目录结构中：

```
wallpaper/
├── desktop/     # 电脑壁纸
├── mobile/      # 手机壁纸
└── avatar/      # 头像
```

每个系列下都有分类和子分类：

```
wallpaper/[series]/[category]/[subcategory]/
```

例如：
- `wallpaper/desktop/动漫/原神/`
- `wallpaper/mobile/人像/氛围感/`
- `wallpaper/avatar/萌宠/猫咪/`

## 命名规范

### 1. 文件名格式

对于自定义壁纸，建议使用以下命名格式：

```
[描述性名称].[扩展名]
```

例如：
- `星空夜景.jpg`
- `猫咪睡觉.png`
- `山水风景.webp`

### 2. 支持的格式

- JPG/JPEG
- PNG
- WebP
- GIF

## 上传步骤

### 1. 选择合适的系列和分类

根据壁纸类型选择对应的系列（desktop/mobile/avatar），然后选择合适的分类和子分类。

### 2. 上传图片

将图片上传到对应的目录中：

```
wallpaper/[series]/[category]/[subcategory]/[文件名]
```

例如：
- 电脑壁纸：`wallpaper/desktop/风景/星空/星空夜景.jpg`
- 手机壁纸：`wallpaper/mobile/萌宠/猫咪/猫咪睡觉.png`
- 头像：`wallpaper/avatar/人像/氛围感/氛围感头像.webp`

### 3. 运行本地处理脚本

上传图片后，需要运行本地处理脚本来更新时间戳、处理 metadata 并生成前端数据文件。

## 注意事项

1. **保持目录结构**：严格按照 `wallpaper/[series]/[category]/[subcategory]/` 的结构组织文件
2. **避免重名**：确保文件名不与现有文件重复
3. **合理分类**：选择最合适的分类和子分类
4. **分辨率建议**：
   - Desktop：1920x1080 或更高
   - Mobile：1080x1920 或更高
   - Avatar：512x512 或更高
5. **文件大小**：建议控制在 10MB 以内，以确保加载速度

## 与原作者更新兼容

1. **不修改核心脚本**：不要修改 `scripts/` 目录下的脚本文件
2. **只添加不删除**：只添加自己的壁纸，不要删除原有的壁纸
3. **定期同步**：定期从原作者仓库同步更新
4. **独立分类**：如果可能，使用独立的分类来存放自己的壁纸

## 常见问题

### Q: 上传后网站不显示新壁纸怎么办？
A: 请确保运行了本地处理脚本，该脚本会更新 metadata 并生成前端数据文件。

### Q: 如何创建新的分类？
A: 直接在对应的系列目录下创建新的分类文件夹即可，系统会自动识别。

### Q: 壁纸分辨率会影响显示吗？
A: 系统会自动检测分辨率并生成相应的标签，建议使用较高分辨率的图片以获得更好的显示效果。
